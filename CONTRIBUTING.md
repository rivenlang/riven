🤝 Contributing to Riven

Thank you for your interest in contributing to Riven Programming Language.

Riven is a long-term open-source project focused on building a modern programming language completely from scratch.

Whether you are fixing bugs, improving documentation, adding tests, or implementing new compiler features, every contribution is appreciated.

---

Before You Start

Please make sure you:

- Read the project documentation.
- Check existing issues before creating a new one.
- Search for existing pull requests.
- Follow the project's coding style.

---

Getting Started

1. Fork the repository.
2. Clone your fork.

git clone https://github.com/your-username/riven.git

3. Create a new branch.

git checkout -b feature/my-feature

4. Make your changes.

5. Test your changes.

6. Commit your work.

git commit -m "Add lexer keyword support"

7. Push your branch.

git push origin feature/my-feature

8. Open a Pull Request.

---

Coding Guidelines

Please follow these rules:

- Write clean and readable C++.
- Prefer modern C++ features when appropriate.
- Keep functions focused on one responsibility.
- Write meaningful variable names.
- Add comments where necessary.
- Do not introduce unnecessary dependencies.

---

Compiler Architecture

When contributing to the compiler:

- Keep the Lexer independent.
- Keep the Parser independent.
- Avoid circular dependencies.
- Follow the existing project structure.
- Keep modules modular.

---

Testing

Before submitting a Pull Request:

- Build the project successfully.
- Ensure existing functionality still works.
- Add tests when introducing new features.

---

Documentation

Documentation improvements are always welcome.

Examples include:

- Tutorials
- Compiler internals
- Language documentation
- Examples
- API documentation

---

Reporting Bugs

When reporting a bug, include:

- Operating System
- Compiler Version
- Steps to reproduce
- Expected behavior
- Actual behavior

---

Feature Requests

Feature requests should include:

- Motivation
- Use cases
- Proposed syntax (if applicable)
- Possible implementation ideas

---

Code Style

Please keep code:

- Simple
- Consistent
- Readable
- Maintainable

---

Thank You

Thank you for helping build Riven.

Every contribution, no matter how small, helps improve the project.
